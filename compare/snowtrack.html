What happened to Snowtrack?

Perforce acquired Snowtrack in March 2025 and rebranded the product as P4 One. The standalone Snowtrack app is no longer available — new customers must subscribe to Perforce's enterprise licensing starting at roughly $19 per seat per month, with the product now positioned as part of the Helix version-control ecosystem rather than a lightweight tool for independent creators.

Why people switch

  • → Enterprise pricing. P4 One starts at roughly $19 per seat per month, pricing out solo artists and small studios who used Snowtrack for free.
  • → Mandatory Perforce account + SSO. New licenses require signing up for Perforce's auth system — not friendly to creators who just want to manage their Figma or Blender files.
  • → Helix Server dependency. The P4 One workflow is built around a central Helix Server — enterprise infrastructure that's overkill for a freelancer or a 3-person studio.
  • → Your files locked in a server. P4 One stores versions inside Helix; you can't just browse your NAS and see your project files. Keeply keeps everything as regular files you can open in Explorer or Finder.
  • → Lost the lightweight-creative feel. P4 One's roadmap clearly leans toward large studios, away from the independent-creator sweet spot Snowtrack used to occupy.

How to switch to Keeply

  1. 1

    Export your Snowtrack history while you still can

    If your Snowtrack install still works, browse to your project folder and copy out the full .snowtrack/ directory to an external drive. Keep the legacy history as a read-only archive.

  2. 2

    Install Keeply and point it at your project folder

    Download Keeply, open your project folder, and click Start protecting. Keeply begins versioning from the current state — fresh history, same files, no migration tool required.

  3. 3

    Pick a backup location (NAS / USB / Gitea / GitHub)

    In Keeply → Backup Locations, add one of: local folder, USB drive, NAS (SMB path), self-hosted Gitea, or a GitHub repo. Keeply syncs there automatically on every save — no central server required.

Frequently asked questions

What happened to Snowtrack? ▼
Perforce acquired Snowtrack in March 2025 and rebranded it as P4 One. The original standalone product is no longer sold — new licenses require a Perforce Helix Server subscription with enterprise pricing starting around $19 per seat per month.
Is Keeply really free? ▼
Yes — the shipping product is free forever for personal use. Core features including unlimited version history, visual diff, file comparison, notes, and backup to one location are free. A Team plan with admin controls and multi-location sync is in active development. A $599 perpetual Founding Member license (500 seats) buys lifetime access to all future Team features when they ship.
Do I need Git knowledge to use Keeply? ▼
No. Keeply is designed specifically for creators who don't want to think about Git — it's a firm product principle that zero Git terminology appears in the interface. Actions use office-friendly language: Save a version, Browse history, Restore to this point, Compare versions. No commits, branches, or merges to learn.
Can Keeply handle large files like 3D, video, or Blender projects? ▼
Yes, and you don't configure anything. Keeply automatically detects 38 binary file formats (Blender, PSD, video, 3D, archives) plus any file over 10 MB, then routes them through large-file storage silently. Repository sizes stay manageable; you never see the Git LFS setup dance that other tools expose.
What file types get visual side-by-side diff? ▼
Keeply renders true visual diffs for Microsoft Office documents (Word .docx, Excel .xlsx, PowerPoint .pptx), PDFs, and raster images (PNG / JPG / WebP). You see two versions side by side with highlighted changes — not just a list of modified bytes. For all other file types, Keeply tracks the version and lets you restore any point.
Where does my data actually live? ▼
Your project files stay on your computer, and backup copies go to one of five self-hosted locations you choose: a local folder, USB drive, NAS (SMB path), self-hosted Gitea, or a GitHub repository. Keeply runs no servers that touch your project data — we only receive a hashed device identifier during license validation.
