1. 本ソフトウェア

Keeply はプロジェクトファイルの視覚的なバージョン管理を提供するデスクトップアプリケーションです。コンピュータ上でローカルに動作し、すべてのデータをローカルデバイスまたは NAS に保存します。Keeply は Git 技術を基盤としていますが、仕事でドキュメントを編集するすべてのユーザー向けに設計されており、Git を直接操作する必要はありません。

当社ウェブサイト、通信、または第三者の決済フローにおいて「購入」「販売」「買う」等の用語が使用されている場合でも、本契約はいかなる所有権も付与するものではありません。

3.1 無料プラン

無料プランは、完全なバージョン管理機能、無制限のユーザー数を永久無料で提供します。無料プランにはチーム管理機能（メンバー管理、権限設定、チームアクティビティログなど）は含まれません。無料プランは、サービスレベルの保証やテクニカルサポートの保証なしに「現状のまま」で提供されます。

Keeplyは、無料プランの機能や提供を随時変更する権利を留保します。

3.2 チームサブスクリプションプラン

チームプランは、ユーザーあたりの課金ではなく定額制です：

  • 月額：$25/月
  • 年額：$250/年

一律価格、ユーザーあたりの課金なし。無料プランのすべての機能に加え、メンバー管理と権限設定、チームアクティビティログ、優先テクニカルサポートが含まれます。

サブスクリプションは、キャンセルしない限り各請求サイクルの終了時に自動更新されます。キャンセルは現在の請求サイクルの終了時に有効となり、それまでサービスを引き続きご利用いただけます。サブスクリプション料金は、第4条に記載の場合を除き、返金不可です。

3.3 永久ライセンス（早期アクセス限定オファー）

永久ライセンスは、早期アクセス限定オファーとして$599でご利用いただけます。500本限定です。一括払いで、購入時のバージョンを永久にご利用いただけます。

主な詳細：

  • 購入時に利用可能なバージョンを無期限にご利用いただけます
  • 永久ライセンスには、将来の機能アップデートやメジャーバージョンアップグレードは含まれません
  • バグ修正アップデートはKeeplyの裁量で提供される場合がありますが、保証はされません
  • Keeplyは、本オファーをいつでも中止または変更する権利を留保します
  • 中止前に購入された永久ライセンスは、元の条件に基づき有効です
  • 新機能への早期アクセスおよび専用コミュニティチャンネル

3.4 永久フォールバックライセンス（1年後のセーフティネット）

1年間の継続サブスクリプション後、いつでもキャンセルできます。キャンセル後もKeeplyは引き続き動作します — 新しいバージョンを受け取れなくなるだけです。新機能が必要になったら、再度サブスクリプションに加入するだけです。

条件：中断や未払いのない12ヶ月間の継続サブスクリプション。

  • ✅ 新バージョンが必要になったら、再度サブスクリプションに加入するだけ

  • 12ヶ月前にキャンセル → 現在の請求サイクル終了時にアクセス終了
  • 中断または未払いがあった場合 → 12ヶ月のカウントがリセット

3.5 定義

「チームボールト」とは、中央のコラボレーションポイントとして指定された共有ストレージの場所を意味します。これには、ネットワーク接続ストレージ（NAS）デバイス、ファイルサーバー、セルフホスト型 Git サーバー、またはクラウドホスト型 Git リモートリポジトリが含まれますが、これらに限定されません。チームボールトは、ソフトウェアによって生成され、リポジトリ内に保存される一意の場所トークンによって識別されます。

「ボールトの場所」とは、チームボールトに関連付けられた一意の識別子を意味し、リポジトリに埋め込まれた場所トークンおよび/または正規化された Git リモート URL から派生します。ボールトの場所は次の 2 種類に分類されます：

  • (a) 「固定の場所」とは、チーム管理者が設定し、すべてのチームメンバーが共有するボールトの場所を意味し、最大 2 つまでです。
  • (b) 「フレキシブルな場所」とは、チームメンバーが申請し、管理者がそのメンバーの個人使用のために承認したボールトの場所を意味し、メンバーごとに最大 1 つです。

3.6 ボールトの場所に関する規則

固定の場所の変更には、以下の条件が適用されます：

  • (a) 直近の変更から 7 日間のクールダウン期間；
  • (b) 年間 6 回以下の変更；
  • (c) リポジトリデータ（埋め込みの場所トークンを含む）を新しいストレージハードウェアに移行することは、場所トークンが変更されない限り、固定の場所の変更とはみなされません。

フレキシブルな場所は、メンバーがソフトウェアを通じて申請し、管理者の承認により有効となります。管理者はいつでもメンバーのフレキシブルな場所を取り消すことができます。30 日間連続してアクセスされなかったフレキシブルな場所は自動的に解放されます。
