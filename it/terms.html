3.5 Definizioni

"Team Vault" indica una posizione di archiviazione condivisa — inclusi, a titolo esemplificativo, un dispositivo Network Attached Storage (NAS), un file server, un server Git self-hosted o un repository Git remoto nel cloud — designata come punto centrale di collaborazione. Un Team Vault è identificato da un token di posizione univoco generato dal Software e memorizzato nel repository.

"Posizione Vault" indica l'identificatore univoco associato a un Team Vault, derivato dal token di posizione incorporato nel repository e/o dall'URL Git remoto normalizzato. Le Posizioni Vault si classificano in due tipi:

  • (a) "Posizione Fissa" indica una Posizione Vault impostata dall'amministratore del team e condivisa da tutti i membri del team, fino a un massimo di 2;
  • (b) "Posizione Flessibile" indica una Posizione Vault richiesta da un membro del team e approvata dall'amministratore per l'uso personale di quel membro, fino a 1 per membro.

3.6 Regole sulle Posizioni Vault

Le modifiche alle Posizioni Fisse sono soggette a quanto segue:

  • (a) Un periodo di attesa di sette (7) giorni dalla modifica più recente;
  • (b) Non più di 6 modifiche all'anno;
  • (c) La migrazione dei dati del repository (incluso il token di posizione incorporato) su nuovo hardware di archiviazione non costituisce una modifica di Posizione Fissa, a condizione che il token di posizione rimanga invariato.

Le Posizioni Flessibili vengono richieste dai membri tramite il Software ed entrano in vigore con l'approvazione dell'amministratore. L'amministratore può revocare la Posizione Flessibile di qualsiasi membro in qualsiasi momento. Le Posizioni Flessibili non utilizzate per 30 giorni consecutivi vengono rilasciate automaticamente.
