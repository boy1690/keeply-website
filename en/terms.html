3.5 Definitions

"Team Vault" means a shared storage location — including but not limited to a Network Attached Storage (NAS) device, file server, self-hosted Git server, or cloud-hosted Git remote repository — designated as the central collaboration point. A Team Vault is identified by a unique location token generated by the Software and stored within the repository.

"Vault Location" means the unique identifier associated with a Team Vault, derived from the repository's embedded location token and/or the normalized Git remote URL. Vault Locations are classified into two types:

  • (a) "Fixed Location" means a Vault Location set by the team admin and shared by all team members, up to a maximum of 2;
  • (b) "Flexible Location" means a Vault Location requested by a team member and approved by the admin for that member's personal use, up to 1 per member.

3.6 Vault Location Rules

Changes to Fixed Locations are subject to the following:

  • (a) A cool-down period of seven (7) days from the most recent change;
  • (b) No more than 6 changes per year;
  • (c) Migration of repository data (including the embedded location token) to new storage hardware does not constitute a Fixed Location change, provided the location token remains unchanged.

Flexible Locations are requested by members through the Software and take effect upon admin approval. The admin may revoke any member's Flexible Location at any time. Flexible Locations not accessed for 30 consecutive days are automatically released.
