1. De Software

Keeply is a desktop application that provides visual version control for project files. It runs locally on your computer and stores all project data on your local device or NAS. Keeply does not operate a user account service — software activation is handled through a License Key mechanism, with the license bound to devices you explicitly designate.

3.5 Definities

"Teamkluis" betekent een gedeelde opslaglocatie — inclusief maar niet beperkt tot een Network Attached Storage (NAS)-apparaat, bestandsserver, zelf-gehoste Git-server of cloudgebaseerde Git-remote repository — aangewezen als het centrale samenwerkingspunt. Een Teamkluis wordt geïdentificeerd door een uniek locatietoken gegenereerd door de Software en opgeslagen in de repository.

"Vault Location" means the logical location marker associated with a Team Vault, derived from the repository's embedded location token and/or the normalized remote URL. Vault Locations are classified into the following types:

  • (a) "Canonical" means the Vault Location designated as the source of truth for team versions; exactly one (1) per Team Vault;
  • (b) "Shared Mirror" means a Vault Location that is a full snapshot synchronized from the Canonical and shared across the team; at most one (1) per Team Vault;
  • (c) "Release" means a version snapshot produced from the Canonical at a specific point in time; unlimited in number;
  • (d) "Clone" means a local working copy derived from the Canonical on a member's device; unlimited in number.

The data form of a Vault Location may be selected at creation time: "Standard mode" allows direct browsing and editing of the Canonical via the file explorer, and the Canonical may be modified by any user with access to that folder; "Protected mode" uses a protected repository format, where the Canonical contents may only be modified through the Software interface and destructive history-changing operations are rejected. Both Standard and Protected modes are available on the Free and Team plans. Personal workflows (version history, file diff, Release publishing, unlimited users, etc.) are identical between the two plans; the Team plan's distinguishing features are team governance tools, not increased capacity or additional locations.

3.6 Regels voor Kluislocaties

Changes to a Shared Mirror Location are subject to the following (applicable only to the Team plan):

  • (a) A cool-down period of seven (7) days from the most recent change taking effect (a period during which no further changes may be made);
  • (b) No more than six (6) changes within any rolling 365-day period;
  • (c) Migration of repository data (including the embedded location token) to new storage hardware does not constitute a Shared Mirror Location change, provided the location token remains unchanged.

When a Shared Mirror Location has not been accessed for ninety (90) consecutive days, the Software will notify the admin, but will not automatically release the location.

Vault Location changes on the Free plan are not subject to cooldowns or annual limits; the foregoing limits apply to the Team plan only.
