Keeply 是一款為專案檔案提供視覺化版本控制的桌面應用程式。它在你的電腦上本地運行，並將所有資料儲存在你的本地裝置或 NAS 上。Keeply 基於 Git 技術構建，但專為設計師、建築師和營建專業人員設計，使用者無需直接與 Git 互動。

3.5 定義

「團隊保管庫」係指由使用者指定作為協作據點的共享儲存位置，包括但不限於網路附接儲存裝置（NAS）、檔案伺服器、自建 Git 伺服器或雲端託管 Git 遠端儲存庫。團隊保管庫以軟體產生並儲存於儲存庫內的唯一位置識別碼加以辨識。

「保管庫位置」係指與團隊保管庫相關聯的唯一識別碼，由儲存庫內嵌的位置識別碼及／或正規化之 Git 遠端 URL 衍生而得。保管庫位置分為兩類：

  • (a)「固定位置」係指由管理員設定、全團隊共用的保管庫位置，最多 2 個；
  • (b)「彈性位置」係指由團隊成員申請、經管理員核准後供該成員個人使用的保管庫位置，每位成員最多 1 個。

3.6 保管庫位置規則

固定位置之變更須遵守以下規定：

  • (a) 自最近一次變更起須經過七（7）日之冷卻期；
  • (b) 每年變更次數不得超過 6 次；
  • (c) 將儲存庫資料（含內嵌位置識別碼）遷移至新的儲存硬體，於位置識別碼未變更之前提下，不構成固定位置之變更。

彈性位置由成員透過軟體申請，經管理員核准後生效。管理員得隨時撤銷任何成員的彈性位置。彈性位置連續 30 天未被存取時，將自動釋放。

3.7 單一實體要求

每組授權僅供單一法律實體（公司、合夥、獨資或同等組織）及其授權承包商使用。多個無關之法律實體不得共享單一授權，即使其共用實體儲存設施亦同。
