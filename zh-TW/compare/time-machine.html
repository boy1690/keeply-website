對災難很厲害。對日常找舊版本不太行。

Time Machine 擅長它被設計的事情：Mac 掛了，插上備份硬碟還原整個系統。但它把每個檔案當成不具名的二進位 blob 快照整顆硬碟。想找某個專案檔案的某個舊版本，要打開 Time Machine、在時光星軌裡飛、猜哪個小時有你想要的版本、然後把整個資料夾還原——因為單檔案還原很脆弱。沒有比對——兩個 PSD 擺在一起你不知道差在哪。Windows 用戶？完全沒這東西。

為什麼要換

  • → 只支援 macOS。 團隊裡一出現 Windows 使用者，Time Machine 就不再是團隊的答案。你需要的是跨作業系統都能用的流程。
  • → 沒有語意理解。 Time Machine 把 200 MB 的 PSD 和 2 KB 的文字檔當成一樣的東西：不透明的 blob。它沒辦法告訴你兩個版本差在哪。你看著兩個長得一樣的檔案，完全無法判斷。
  • → 還原是整個資料夾層級的事。 要只拿一個檔案的一個舊版本、又不想蓋掉其他東西，真的很麻煩。很多人最後就是把舊資料夾複製到別的地方——製造新的混亂。
  • → 跟整顆硬碟的所有東西混在一起。 Time Machine 備份你的下載資料夾、系統檔案、整台 Mac。要找某個客戶專案的主視覺舊版本，要先從這一堆裡撈。

怎麼換到 Keeply

  1. 1

    Time Machine 繼續跑——它的災難還原就是好用

    別關掉。Time Machine 對「Mac 整台掛了」的情境是正當的保險。Keeply 管日常版本找回，是不同的工作。

  2. 2

    在需要真實版本歷史的專案裡裝 Keeply

    挑你手上的客戶資料夾、設計檔、每天在改的作品。用 Keeply 打開、點「開始保護」。從目前狀態開始管版本，存檔時你自己寫附註。

  3. 3

    為團隊成員加一個共用備份目標

    多人專案，在 Keeply 裡設定 NAS、Gitea、或 GitHub 備份。每個人的版本歷史匯集到整個團隊都能拿到的地方——這是 Time Machine 做不到的事。

  4. 4

    Time Machine 負責災難，Keeply 負責日常找回

    硬碟掛了？Time Machine。客戶問三週前 Logo 長什麼樣？Keeply 時間軸，一點、視覺比對，搞定。

常見問題

可以同時跑 Time Machine 和 Keeply 嗎？ ▼
可以，而且我們推薦這樣。Time Machine 對整台 Mac 做災難還原。Keeply 以專案為單位管版本歷史，附筆記和視覺比對。它們解決不同問題、不會衝突——Keeply 的資料放在你專案資料夾裡，Time Machine 照常把它跟其他東西一起備份。雙保險。
Keeply 支援 Windows 嗎？我的團隊混合使用。 ▼
支援。Keeply 在 Windows 和 macOS 都能跑，所以混合團隊共用同一個工具、同一份版本歷史、同一個備份目標。Linux 在 roadmap 上但還沒出貨。你的團隊今天有人用 Windows 的話，光靠 Time Machine 完全覆蓋不到——Keeply 可以。
要找單一檔案的特定舊版本怎麼辦？ ▼
這是最大差別。Time Machine 要打開時光星軌、猜哪個小時的快照有你要的版本、還原。Keeply 是打開專案、點那個檔案、在時間軸上看每次存檔附自己的筆記、點要的版本就到。找一個月前的版本是幾秒鐘的事。
Keeply 能告訴我兩個版本差在哪嗎？ ▼
可以。Keeply 對 Word、Excel、PowerPoint、PDF、圖片都能並排視覺比對——直接告訴你哪段文字、哪個數字、哪個像素不一樣。Time Machine 沒有這種功能；它把每個檔案當成不透明二進位，差異要你自己找。對設計或文件工作來說，比對才是你真正需要的東西。
我需要另外一顆像 Time Machine 要的外接硬碟嗎？ ▼
不用。Keeply 把版本放在你專案資料夾裡，只存差異，空間成本遠小於完整檔案。你可以加備份目標（隨身碟、NAS、GitHub、Gitea）但不是必要的，一開始不需要。一個有好幾個月歷史的 Keeply 專案常常一根隨身碟就裝得下，Time Machine 要整顆硬碟。
我的版本實際存在哪裡？ ▼
預設放在你自己的儲存空間：在你專案資料夾裡，以差異方式高效儲存。你可以加最多三個備份位置：辦公室 NAS（SMB）、隨身碟、自架 Gitea、或私有 GitHub repo。Keeply 自家伺服器完全不存你的專案資料。隨時不用或移除，檔案和版本歷史都留在原地。
