為什麼要換

  • → 版本歷史藏在三層選單後面。 「檔案 → 版本記錄 → 查看版本記錄 → 一個一個點開看」。一週做十次，永遠不會變快。
  • → 沒辦法比對到底改了什麼。 Drive 讓你看舊版本的預覽，但看不出哪段文字、哪個數字、哪個像素跟現在的檔案不一樣。只能兩個都開、切換分頁、用猜的。
  • → 30 天 / 100 版本上限。 Drive 會默默刪掉舊版本，除非你記得把每一個都標記為「永久保留」。活躍專案天天改，幾週就撞牆。
  • → 離線？版本歷史完全消失。 版本資料在 Google 雲端。飛機上、客戶端沒 Wi-Fi、Drive 掛掉的時候，歷史就是完全拿不到。

怎麼換到 Keeply

  1. 1

    把 Drive 資料夾同步到本機

    安裝 Google Drive for Desktop，讓它把工作資料夾鏡像到本機。Keeply 需要磁碟上的真實檔案才能管版本——線上專用的文件不行。

  2. 2

    安裝 Keeply，打開本機的 Drive 資料夾

    下載 Keeply，開啟同步下來的專案資料夾，點「開始保護」。從此每次存檔都進時間軸，附上你寫的附註。

  3. 3

    把 Google 文件匯出為真實格式

    想好好管版本的 Docs / Sheets / Slides，用「檔案 → 下載」存成 .docx / .xlsx / .pptx。Keeply 的視覺比對對這些真實檔案有效，對 Google 的專有格式無效。

  4. 4

    新增一個你掌握的備份位置

    在 Keeply 裡新增 NAS、隨身碟、或 GitHub repo 作為備份目標。版本歷史從此放在你擁有的儲存空間——Google 掛了、上限到了都影響不到它。

常見問題

可以繼續用 Google Drive 分享檔案、同時用 Keeply 嗎？ ▼
可以，它們解決不同問題。Drive 即時分享檔案給協作者；Keeply 在本機追蹤工作檔案的版本歷史。把 Drive 同步到桌面，Keeply 指向同步下來的資料夾，兩者平行運作。分享交給 Drive，歷史交給 Keeply。
Keeply 支援 Google Docs、Sheets、Slides 嗎？ ▼
要先下載成真實檔案格式（.docx / .xlsx / .pptx）才行。Google 原生的 Docs 格式只存在 Google 伺服器上，Keeply 看不到。想長期保存的文件記得定期「檔案 → 下載」——Keeply 會接手管版本並提供視覺比對。
為什麼說 Drive 的版本歷史有限？ ▼
三個理由。預設保留 30 天或 100 個版本，先到先刪。沒有並排比對——只能預覽某個版本。非 Google 格式的歷史就是一堆沒附註的不透明快照。Keeply 三個都解：無限保留、真正的視覺比對、每次存檔要你寫附註。
我需要停掉 Workspace 嗎？ ▼
完全不用。Workspace 的 Email、共用雲端硬碟、Meet 還是值得的錢。用 Keeply 免費版換掉版本歷史那一塊就好。不再為一個用一個月就嫌小的功能每人每月付錢，也完全不影響 Workspace 的協作價值。
離線怎麼辦？ ▼
Keeply 完全離線可用——存檔瞬間就寫進本機磁碟，不管有沒有網路。連線恢復後，你設定的備份位置（NAS、GitHub、Gitea）會自動同步。Drive 剛好相反，讀寫版本歷史都要連線，一上飛機就掛。
我的版本實際存在哪裡？ ▼
放在你自己的儲存空間。Keeply 把版本資料寫進你專案裡的隱藏資料夾（本機磁碟），再選擇性鏡像到你挑的備份位置：辦公室 NAS（SMB）、隨身碟、自架 Gitea、或私有 GitHub repo（最多三個目的地）。Keeply 自家伺服器完全不存你的專案資料。隨時不用，檔案和歷史都留著。
