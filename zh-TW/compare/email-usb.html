每週五要靠你記得的備份，不是備份

寄信 + 隨身碟的儀式能用，是因為你執行它。你跳過的那一週——壓期、出差、忘了——就是筆電摔下去、隨身碟還插在抽屜裡的那一週。附件大小限制（Gmail 25 MB、Outlook 20 MB）剛好過濾掉你最想備份的設計檔和影片草稿。隨身碟會實體壞掉、會遺忘在咖啡廳。寄出的信裡五份「final_proposal.docx」擺在寄件匣，也沒有任何版本結構告訴你哪一個才是真的 final。Keeply 自動化每次存檔的版本管理和備份到你擁有的位置——你不再是單點故障。

為什麼要換

  • → 你每週都要記得。 你忘記的那週，就是你需要它的那週。生病、出差、重要交件——儀式就是在這些時候斷掉，筆電也偏偏挑這些時候掛。永遠靠自己的紀律不是備份策略。
  • → 附件大小限制剛好卡住最重要的檔案。 Gmail 25 MB。Outlook 20 MB。一般 PSD、一段影片草稿、含嵌入物的大 PDF——就是這些。你最想備份的那些檔案，儀式剛好護不到。
  • → 隨身碟用你沒想到的方式壞掉。 口袋裡被壓壞、忘在咖啡廳、借走沒還、在最糟糕的時候摔到。隨身碟是沒有冗餘的單點故障——跟備份該做的事剛好相反。
  • → 寄件匣裡五個副本不是版本歷史。 寄件匣裡有 final.docx、final_v2.docx、final_real.docx、final_sent.docx。哪一個是客戶收到的？哪一個有更新的定價？Email 不知道。三週後你也不知道。

常見問題

寄信給自己真的那麼不可靠？ ▼
它有用，直到它沒用，而且失效模式都很糟。附件上限（Gmail 25 MB、Outlook 20 MB）默默擋掉你最大的檔案。信件服務商有時會把重複附件標成垃圾信。最重要的是，它靠你每週記得，一做做好幾年。儀式當成額外一層保險沒問題，但當主要備份，它剛好在你最需要的時候失效。
Keeply 能完全取代我的隨身碟嗎？ ▼
可以，而且應該，因為單一隨身碟是單點故障。Keeply 允許你設定最多三個備份目的地：本機、隨身碟、NAS、Gitea、GitHub 任意組合。典型設定是本機 + NAS + 私有 GitHub repo，自動拿到三份獨立副本。想繼續帶隨身碟當偶爾的額外副本也行——它只是不再是你和弄丟筆電之間唯一的東西。
如果我大多時候離線、沒辦法同步雲端備份怎麼辦？ ▼
Keeply 存檔瞬間就在本機管版本，不管有沒有網路。你設定的備份目標（NAS、GitHub、Gitea）在連上網的時候機會性同步。在飛機上、偏遠客戶端、ISP 掛的時候，版本管理照常跑——你只是暫時只有一份副本而不是三份。重新上線時備份會自己追上。
我有超大的影片檔。Keeply 處理得了嗎？ ▼
可以。Keeply 會默默偵測二進位格式（38 種檔案類型，包括 PSD、AI、Sketch、MP4、MOV、ZIP 等等）加上任何超過 10 MB 的檔案，底層用差異儲存高效管版本。你不用設定、不用標記——就是會動。沒有 25 MB 附件上限、不用手動設 LFS、大檔案不用另外的工具。
單人營運、沒有 IT 支援的情境能用嗎？ ▼
那正是它被打造的對象。沒有要跑的伺服器、沒有命令列、不需要 Git 知識。安裝、打開資料夾、點「開始保護」。想加備份目的地（你的隨身碟、你的 NAS、你自己的 GitHub 帳號）就在一個簡單的設定畫面裡設定。整個安裝花不到十分鐘，之後不需要每週的紀律。
我的版本歷史實際存在哪裡？ ▼
放在你自己的儲存空間。Keeply 把版本寫進你專案裡的隱藏資料夾（Windows 或 macOS 本機磁碟），再加上你設定的最多三個備份目的地：辦公室 NAS（SMB）、隨身碟、自架 Gitea、或私有 GitHub repo。Keeply 自家伺服器完全不存你的專案資料。隨時停止使用，檔案和歷史都留在原地。
