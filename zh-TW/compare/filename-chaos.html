為什麼要換

  • → 你記不得 v3、v4、真的最終版到底差什麼。 三週後檔名只告訴你先後順序，不告訴你內容。打開來看——90% 長得一樣，差異藏得好好的。
  • → 不小心蓋檔是日常。 你本來想存成 cover_v5.psd。Cmd+S 按下去是習慣動作。現在 cover_FINAL.psd——客戶核准的那個——變成你沒做完的版本。回不去。
  • → 「寄週二那版給我。」 哪個週二？你查檔案修改日期、對照 Email 來回翻，還是在猜。你需要的資訊從來沒有以可用的形式存在過。

怎麼換到 Keeply

  1. 1

    挑檔名最亂的那個資料夾

    通常是你最活躍的專案——最多「真的最終版_v2」的那個。別想一次整理所有資料夾；從痛最深的地方開始。

  2. 2

    安裝 Keeply，指向那個資料夾

    下載 Keeply、打開資料夾、點「開始保護」。Keeply 把目前狀態當作第一個版本開始追蹤。你不用改名、不用重整。

  3. 3

    停止再加 _v2、_FINAL、_真的最終版 後綴

    檔名維持乾淨——cover.psd，不是 cover_真的最終版.psd。每次存檔進時間軸，你寫一句短筆記（「客戶核准」、「加了定價框」、「週二交件」）。筆記才是檔名原本假裝要做的事。

  4. 4

    要找舊版本時看時間軸，不要看檔案總管

    下次要找「週二那版」或「加定價框之前」，打開 Keeply 時間軸，不要翻檔名。一點、視覺比對、還原或匯出。肌肉記憶一週就練起來。

常見問題

改檔名不就是免費的嗎？為什麼要付錢？ ▼
個人版永久免費，所以錢不是問題——問題是檔名混亂是不是默默在吃你的時間。如果你曾花 20 分鐘找「對的版本」、寄錯檔案給客戶、或不小心蓋掉客戶核准的版本，答案就是有。美金成本是零；時間成本才是 Keeply 幫你省的。
我要完全放棄我的命名規則嗎？ ▼
不用，而且我們建議轉換的前幾週先保留。你舊的 cover_v1.psd 到 cover_真的最終版_修正版.psd 照樣留在原地。Keeply 從今天開始用一個乾淨檔名建新的時間軸。用時間軸用幾週後，你會發現自己不再伸手去加後綴——那個舊規則原本要解決的問題，Keeply 已經接手了。
如果我真的想要多個平行變體，不是線性歷史呢？ ▼
還是可以。cover.psd 和 cover_alt.psd 真的是不同變體時就當兩個真實檔案；Keeply 各自管它們的歷史。v1 / v2 / FINAL 這個模式是專門用來記同一個檔案的時間變化——那就是時間軸要取代的事情。平行變體是不同概念，它們還是可以有真實的檔名。
我只是要快速存個小修正，會不會變慢？ ▼
不會。Keeply 接在存檔動作上，額外成本接近零。每次存檔只存差異，不是整個檔案複製，磁碟成本很小。唯一多的一件事是你寫一句短筆記——一句話——說明這次存檔在做什麼。這一句話，就是「cover_v17.psd」和一年後你能操控的時間軸之間的差別。
非技術背景的同事看得懂嗎？ ▼
看得懂——這是我們設計時的硬性要求。完全不需要 Git 知識、不用命令列、沒有 branch 或 merge 的概念。你打開專案、你存檔、Keeply 管版本。介面上是你自己用自己的話寫的筆記所組成的時間軸。設計師、寫手、接案者、行政同事都不需要工程背景就能用。
我的版本歷史實際存在哪裡？ ▼
放在你自己的儲存空間。Keeply 把版本放在你專案資料夾裡的隱藏資料夾（Windows 或 macOS 本機磁碟），再加上你設定的最多三個備份位置：辦公室 NAS（SMB）、隨身碟、自架 Gitea、或私有 GitHub repo。Keeply 自家伺服器完全不存你的專案資料。檔案本身維持為一般檔案，檔案總管或 Finder 隨時可開。
